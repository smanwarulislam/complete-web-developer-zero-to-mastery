Sass: Syntax

Syntax
Sass supports two different syntaxes. Each one can load the other, so it‚Äôs up to you and your team which one to choose.

SCSS
The SCSS syntax uses the file extension .scss.
With a few small exceptions, it‚Äôs a superset of CSS, which means essentially all valid CSS is valid SCSS as well.
Because of its similarity to CSS, it‚Äôs the easiest syntax to get used to and the most popular.

The Indented Syntax
The indented syntax was Sass‚Äôs original syntax, and so it uses the file extension .sass.
Because of this extension, it‚Äôs sometimes just called ‚ÄúSass‚Äù.
The indented syntax supports all the same features as SCSS,
but it uses indentation instead of curly braces and semicolons to describe the format of the document.

In general, any time you‚Äôd write curly braces in CSS or SCSS, you can just indent one level deeper in the indented syntax.
And any time a line ends, that counts as a semicolon.
There are also a few additional differences in the indented syntax that are noted throughout the reference.

.scss is actually a file format for what we call Sass.
It's kind of like CSS but it gives you extra features.
